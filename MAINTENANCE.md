# Maintenance Mode

**Status: MAINTENANCE ONLY**

## Accepted changes

- Security patches
- Checksum updates (with version bump)
- Documentation corrections
- Signed releases

## Rejected changes

- New features
- New primitives
- Profile schema changes (without major version)
- Failure code additions

## Change process

1. Open issue describing change
2. Reference checksum files affected
3. Create PR with version bump
4. Update all checksums
5. Create signed release

This project has no roadmap.

## No TODO
